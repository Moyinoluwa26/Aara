GET / This Displays the Description to the Services
GET /user/:id This will be used to get Owner's own Posts as it can also be used to view any other Users own Posts . It can be used to display profile Contents
POST /auth/signup This Creates aUser to the Database
POST /auth/login This gets users from the database and hashedPassowrd that will be further validated by comparing it to the suplied body.password. Once, validated , it returns a Token that you use to in the req.header("authorization") and can be stored in whatever facilities you choose from LocalStorage solution , Caching to stateful teachnologies like Redux
GET /post/ This will get all the posts in our database and show them like a Social Media Home feed
GET /post/:id This will take you to the particular Post and It's Information alone . You can use it to display a single Post page and It's Components like, comments, likes et.c
POST /post/ Used to create a post . Here will have pass the Token to the req.header for the middleware to pass the user as valid
PUT /post/:id This is Used to update a single post . Here you pass the pathname the post.id parameter and also verify the Token . It will return a newly updated database element for that post.id and will ne available for the UI on refresh according the state management you Use.
DELETE /post/:id This is used to delete a post(single) . It will also require the psot.id parameter be passed to the route. Changes will reflect in the UI on refresh acording to your state management technology
